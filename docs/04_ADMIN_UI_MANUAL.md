# 04 管理画面使い方マニュアル

## ログイン画面

ログイン画面は入口です。LPのような長い紹介文は入れません。

確認用ボタン：

- オーナーデモで確認
- デモユーザーではじめる

本番では、Firebase Authでメールログイン・Googleログインを使います。

## 初回セットアップ

初回セットアップで以下を選びます。

- 作りたいもの：note連載 / Kindle原稿 / 自分史 / 事業ストーリーなど
- 文章の雰囲気：静かで深い / 映画風 / 前向きなど
- 整理したいテーマ：病気 / 家族 / 借金 / 介護 / 再出発 / AIへの挑戦など

ここで選んだ内容は、管理画面のおすすめ導線に反映されます。

## ダッシュボード

登録済み素材、感情メモ、note記事、Kindle章立て、画像プロンプトの件数を確認します。

オーナーで入った場合は、オーナー無料アカウント表示が出ます。

## 人生素材カード

過去の経験や出来事を、最初は単語や短文で登録します。

入力例：

```text
出来事：17歳で目の病気が発症
時期：17歳
キーワード：不安、病院、将来、見えにくさ
公開レベル：ぼかして使う
今につながる意味：見えにくい人でも使えるAIツールを作りたい
```

## 感情メモ

素材カードに対して、その時の感情・景色・言えなかった言葉を登録します。

ここが小説風の臨場感を作る材料になります。

## 物語生成エンジン

以下の構成方式を選べます。

- ハイブリッド
- 新PASONA重視
- 神話物語重視
- 小説風回想型
- Kindle深掘り型
- ビジネス導線型

標準はハイブリッドです。
新PASONAで読者の共感と行動導線を作り、神話物語で苦境・出会い・試練・再起の流れを作ります。

## note記事生成

記事数はボタンで選びます。

- AIに任せる
- 8記事
- 12記事
- 18記事
- 24記事
- 30記事

noteは、現在の話から入り、途中で回想を差し込み、最後に次回予告でつなぐ構成が基本です。

## Kindle原稿生成

章数はボタンで選びます。

- AIに任せる
- 7章
- 9章
- 12章
- 15章

Kindleはnoteより深く、時系列を整理して一冊の本として再編集します。

## 画像プロンプト

noteヘッダー、記事中画像、Kindle表紙、章扉、SNS画像向けのプロンプトを生成します。

画像は記事と関係のあるものを使うことで、訴求力を上げます。

## 記事管理

記事の状態を管理します。

- 未生成
- 生成済み
- 修正中
- 画像作成済み
- 投稿済み
- Kindle流用候補

## 設定・規約

アカウント・料金、AI設定、規約ドラフトを確認します。
本番では正式な規約ページと同意ログ保存を入れます。
