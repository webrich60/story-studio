# 06 Codex修正ワークフロー手順

## 基本方針

Codexには、いきなり全部修正させません。
最初は必ず「調査だけ」です。

## GitHubアップ後、最初にCodexへ貼る指示

```text
このリポジトリは WEBRICH Story Studio v1.4 です。
まず全体構成を調査してください。
まだコードは変更しないでください。
現在の構成、UI、保存方式、未実装機能、次に実装すべき順番を日本語で整理してください。
```

## 次に依頼する順番

### 1. Firebase Auth調査

```text
Firebase Authを実装する前提で、現在のindex.htmlのどこを分割・修正すべきか調査してください。
まだコードは変更しないでください。
```

### 2. ファイル分割

```text
現在の単体HTMLを、GitHub運用しやすいようにHTML/CSS/JSへ分割してください。
機能や見た目は変えず、構成だけを整理してください。
```

### 3. Firebase Auth実装

```text
Firebase AuthでメールログインとGoogleログインを実装してください。
公開画面にオーナー登録ボタンは出さず、ログイン後のメール判定でowner権限を付与できる設計にしてください。
```

### 4. Firestore保存

```text
localStorage保存をFirestore保存に置き換えてください。
ユーザーごとに人生素材、感情メモ、記事、画像プロンプトを分離保存してください。
```

### 5. AI API接続

```text
AI APIキーはフロントに置かず、Cloudflare WorkersまたはFirebase Functions経由で呼び出す設計にしてください。
Gemini / OpenAIを用途別に切り替えられるようにしてください。
```

## 注意

Codexに「全部本番化して」と一気に依頼しないでください。
1機能ずつ進める方が安全です。
